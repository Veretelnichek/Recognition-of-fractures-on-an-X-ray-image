# Медицинская Диагностика Переломов на Рентгеновских Снимках

Приложение на базе сверточной нейронной сети для классификации и анализа рентгеновских снимков с целью выявления переломов костей. Система использует собственную реализацию CNN без использования высокоуровневых фреймворков типа TensorFlow/Keras.

## Описание проекта

Это десктопное приложение на PyQt5, которое позволяет:
- Загружать рентгеновские снимки (PNG, JPG, BMP и т.д.)
- Классифицировать изображение на основе обученной нейронной сети
- Определять наличие перелома на рентгене
- Анализировать конкретную часть тела, где может быть перелом
- Определять тип перелома

### Архитектура нейронной сети

Модель представляет собой трёхслойную свёрточную нейронную сеть:

**Слой 1 - RGB свёрточный:**
- 3 канала (R, G, B)
- 64 фильтра размером 3×3
- ReLU активация

**Слой 2 - Вторая свёртка:**
- 192 фильтра размером 4×4
- ReLU активация
- Max pooling 2×2

**Слой 3 - Третья свёртка:**
- 384 фильтра размером 3×3
- ReLU активация
- Max pooling 2×2

**Выходной слой:**
- 3 нейрона (классификация на 3 категории)
- Сигмоид активация

**Входное изображение:** 300×300 пиксели

### Классы классификации

1. **NP** - Нет рентгена (изображение не содержит рентгеновского снимка)
2. **NR** - Нет перелома (рентгеновский снимок без переломов)
3. **PR** - Перелом обнаружен (рентгеновский снимок с переломом)

## Использование

### Запуск основного приложения

```bash
python main.py
```

При запуске откроется графическое приложение с интерфейсом для анализа рентгеновских снимков.

**Инструкция по использованию:**

1. Нажмите кнопку **"Загрузка"** для выбора рентгеновского снимка на вашем компьютере
2. Выбранное изображение отобразится в центральной части окна
3. Нажмите кнопку **"Распознавание"** для анализа
4. Результаты будут отображены в информационном блоке внизу окна

Вывод содержит:
- Путь к загруженному файлу
- Время обработки (в секундах)
- Размер обрабатываемого изображения (300×300)
- Результат классификации (тип перелома/отсутствие перелома)
- Дополнительный анализ части тела и типа перелома (если перелом обнаружен)

### Обучение модели

```bash
python Neural_network.py
```

Параметры обучения (можно изменить в коде):
- **Эпохи:** 8
- **Learning rate:** 0.005
- **Размер батча:** 1 (обработка по одному изображению)

Обученные веса сохраняются в формате CSV в папке `weights/`:
- `R_conv_layer.csv` - веса красного канала
- `G_conv_layer.csv` - веса зелёного канала
- `B_conv_layer.csv` - веса синего канала
- `snd_conv_layer.csv` - веса второго слоя свёртки
- `trd_conv_layer.csv` - веса третьего слоя свёртки
- `output_layer.csv` - веса выходного слоя

### Дополнительные скрипты

**`Neural_network_chast_tela.py`** - Модель для определения части тела на рентгене

**`Neural_network_vid_pereloma.py`** - Модель для определения типа перелома

Эти модели работают в паре с основной моделью для детализации анализа.

## Технические детали

### Оптимизация производительности

- **Numba JIT компиляция:** Используется `@njit` для ускорения критичных операций
- **NumPy операции:** Векторизованные вычисления для работы с матрицами
- **CPU обработка:** Приложение работает на CPU (GPU не требуется)

### Характеристики обучения

- **Размер изображения:** 300×300 пиксели
- **Батч:** 1 изображение за итерацию
- **Оптимизатор:** SGD с фиксированным learning rate
- **Функция потерь:** Cross-entropy (сигмоид + MSE)
- **Нормализация:** 0.01-0.99 диапазон для каналов

### Временные характеристики

- **Время вывода:** ~0.1-0.3 сек на одно изображение (зависит от ПК)
- **Время обучения:** Несколько часов на 8 эпохах (зависит от размера датасета)

## Расширение функционала

Для улучшения точности можно:

1. **Увеличить количество слоев** - Добавить больше сверточных слоев
2. **Расширить датасет** - Использовать больше обучающих изображений
3. **Добавить полносвязные слои** - Для более сложных закономерностей
4. **Увеличить размер входа** - Использовать 512×512 или 1024×1024
5. **Увеличить эпохи** - Для лучшей сходимости (текущий: 1-8)
6. **Data augmentation** - Применять трансформации к обучающим данным (ротация, сдвиг, масштабирование)
7. **Использовать GPU** - Перенести вычисления на CUDA для ускорения

## Результаты

Текущая модель демонстрирует хорошую точность при работе с чистыми рентгеновскими снимками. Файлы `efficiency_arr.txt` и `efficiency1_arr.txt` содержат результаты валидации на тестовом наборе.

4. На Linux может потребоваться установка дополнительных пакетов PyQt5

Для отладки вывода нейросети добавьте в `main.py` перед `sys.exit(app.exec_())`:
